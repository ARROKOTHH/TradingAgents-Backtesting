3. 量化趋势过滤器 (Trend Filter)
该过滤器用于确保我们只在市场处于明确的上升趋势时才考虑开仓做多，以过滤掉主要的下跌和震荡行情。

规则 1: 价格趋势确认: 当前收盘价必须高于其长周期指数移动平均线（EMA）。

代码逻辑: self.data.close[0] > self.ema_trend[0]
解释: 这是判断市场牛熊的最常用方法之一。价格在长期均线上方，表明市场多头力量占据主导地位。
规则 2: 均线方向确认: 长周期EMA本身必须处于上升状态。

代码逻辑: self.ema_trend[0] > self.ema_trend[-1]
解释: 此规则能有效过滤掉价格围绕一条走平的均线反复波动的“假趋势”或震荡市，增加了趋势的可靠性。
4. 量化入场信号 (Entry Signal)
在趋势过滤器确认为“上升趋势”的前提下，执行以下入场规则。

主要规则 (Plan A): MACD动能反转信号

逻辑: 捕捉在主升趋势中的健康回调结束，短期动能重新与长期趋势对齐的时刻。
规则组合:
趋势确认: 满足上述【量化趋势过滤器】的所有规则。
回调结束: MACD柱状图（histo）从负值上穿到正值。这表明短期下跌或盘整的动能已经衰竭，多头动能开始回归。
代码逻辑: self.macd.histo[0] > 0 and self.macd.histo[-1] <= 0
强度确认: 在触发信号当天，RSI值必须大于p_rsi_floor（例如40）。
代码逻辑: self.rsi[0] > self.p.p_rsi_floor
解释: 此规则用于过滤掉那些过于深度、可能导致趋势反转的弱势回调。我们希望看到的是强势整理后的再度上攻，而非泥沙俱下的崩盘式回调。
备用规则 (Plan B): 动态支撑位反弹信号

逻辑: 当Plan A的动能信号不明确时，此规则用于捕捉价格回调至短期动态支撑位（如20周期EMA）并获得支撑反弹的经典技术形态。
规则组合:
趋势确认: 满足上述【量化趋势过滤器】的所有规则。
回调触及支撑: 当日最低价曾一度跌破短期EMA（p_pullback_ema），但收盘价最终收回在该均线上方。
代码逻辑: self.data.low[0] < self.ema_pullback[0] and self.data.close[0] > self.ema_pullback[0]
解释: 这个“下影线”形态清晰地表明，在回调过程中多头在关键的动态支撑位进行了有效抵抗，空头力量衰竭，是潜在的买入时机。
5. 量化出场逻辑 (Exit Logic)
止盈/止损规则: 基于ATR的追踪止损 (ATR Trailing Stop)
逻辑: 该机制旨在保护利润并让其奔跑，同时在趋势发生明确反转时及时退出。它比固定止损更优，因为它能根据市场的波动性动态调整止损距离。
执行步骤:
初始止损: 在入场时，计算当时的ATR值。初始止损价位设定为：入场价 - p_atr_multiplier * ATR值。
追踪止损: 在持仓期间，每个交易日重新计算一个新的潜在止损位：当日收盘价 - p_atr_multiplier * ATR值。
更新规则: 如果新的潜在止损位 高于 当前的追踪止损位，则将追踪止损位上调至这个新位置。追踪止损位只上调，不下调。
触发退出: 如果某日的最低价触及或跌破了当前的追踪止损位，立即平仓。
6. 量化风险管理 (Risk Management)
仓位规模: 固定风险百分比模型 (Fixed Percentage Risk Model)
逻辑: 确保任何单笔交易的失败都不会对总资本造成毁灭性打击。无论股价高低、波动大小，每笔交易的预期最大亏损都恒定为总资产的一个固定百分比。
计算公式:
确定单笔风险金额: Risk_Amount = Portfolio_Value * p_risk_percent
确定每股风险: Risk_Per_Share = Entry_Price - Initial_Stop_Loss_Price (初始止损价由ATR规则在入场时确定)
计算仓位规模 (股数): Position_Size = Risk_Amount / Risk_Per_Share
解释: 这种方法将仓位大小与股票自身的波动性（通过ATR止损反映）和您的风险偏好直接挂钩，是专业量化交易中进行风险管理的核心标准。

import backtrader as bt
import math

class CustomStrategy(bt.Strategy):
    """
    策略摘要:
    {st.session_state.report_summary}
    
    该策略结合相对强弱指数(RSI)和移动平均收敛散度(MACD)指标，旨在捕捉超卖后的趋势反转机会。
    - 入场信号: 采用两步确认法。首先，等待RSI进入超卖区，表明市场可能回调过度。一旦此条件满足，策略将进入“等待确认”状态。随后，如果MACD指标发生金叉（MACD线上穿信号线），则视为趋势反转的确认信号，执行买入。
    - 出场信号: 包含三种情况：
        1. 止盈: 价格达到预设的止盈目标。
        2. 止损: 价格跌至预设的止损位。
        3. RSI超买: RSI指标进入超买区，预示上涨动能可能衰竭，主动平仓。
    - 风险偏好: 策略参数（如RSI阈值、MACD周期、止盈止损比例）会根据最终风险偏好（保守型、中等型、激进型）进行动态调整。
    """

    # 默认参数（对应“中等型”风险偏好）
    params = (
        ('risk_appetite', '中等型'),  # 将被 __init__ 中的动态设置覆盖
        ('rsi_period', 14),
        ('rsi_oversold', 30),
        ('rsi_overbought', 70),
        ('macd_fast', 12),
        ('macd_slow', 26),
        ('macd_signal', 9),
        ('stop_loss', 0.07),      # 7% 止损
        ('take_profit', 0.15),    # 15% 止盈
        ('sizing_pct', 0.95),     # 使用95%的现金进行交易
    )

    def __init__(self):
        # --- 1. 根据最终风险偏好动态调整参数 (必须在__init__方法的最开始) ---
        self.p.risk_appetite = "{st.session_state.final_risk_appetite}"

        if self.p.risk_appetite == '激进型':
            self.p.rsi_oversold = 35
            self.p.rsi_overbought = 65
            self.p.macd_fast = 8
            self.p.macd_slow = 18
            self.p.stop_loss = 0.10      # 更宽的止损以承受更大波动
            self.p.take_profit = 0.10    # 更紧的止盈以快速获利
        elif self.p.risk_appetite == '保守型':
            self.p.rsi_oversold = 25
            self.p.rsi_overbought = 75
            self.p.macd_fast = 15
            self.p.macd_slow = 30
            self.p.stop_loss = 0.05      # 更紧的止损以控制风险
            self.p.take_profit = 0.20    # 更宽的止盈以捕捉长期趋势

        # --- 2. 初始化策略变量和数据引用 ---
        self.dataclose = self.datas[0].close
        self.daily_values = []
        self.order = None
        self.buyprice = None
        self.buycomm = None
        
        # 状态管理变量，用于实现“先...后...”的多步逻辑
        self.oversold_signal_active = False

        # --- 3. 初始化指标 ---
        self.rsi = bt.indicators.RSI(self.datas[0], period=self.p.rsi_period)
        self.macd = bt.indicators.MACD(
            self.datas[0],
            period_me1=self.p.macd_fast,
            period_me2=self.p.macd_slow,
            period_signal=self.p.macd_signal
        )
        # 使用CrossOver指标清晰地捕捉金叉信号 (返回+1.0) 和死叉信号 (返回-1.0)
        self.macd_crossover = bt.indicators.CrossOver(self.macd.lines.macd, self.macd.lines.signal)

    def log(self, txt, dt=None):
        """ 策略日志记录函数 """
        dt = dt or self.datas[0].datetime.date(0)
        print(f'{dt.isoformat()} | {txt}')

    def notify_order(self, order):
        if order.status in [order.Submitted, order.Accepted]:
            return

        if order.status in [order.Completed]:
            if order.isbuy():
                self.log(f'BUY EXECUTED, Price: {order.executed.price:.2f}, Cost: {order.executed.value:.2f}, Comm: {order.executed.comm:.2f}')
                self.buyprice = order.executed.price
                self.buycomm = order.executed.comm
            elif order.issell():
                self.log(f'SELL EXECUTED, Price: {order.executed.price:.2f}, Cost: {order.executed.value:.2f}, Comm: {order.executed.comm:.2f}')
            self.bar_executed = len(self)
        elif order.status in [order.Canceled, order.Margin, order.Rejected]:
            self.log('Order Canceled/Margin/Rejected')

        self.order = None

    def notify_trade(self, trade):
        if not trade.isclosed:
            return
        self.log(f'OPERATION PROFIT, GROSS {trade.pnl:.2f}, NET {trade.pnlcomm:.2f}')

    def next(self):
        # --- 核心准则：诊断日志 (每日关键指标) ---
        current_date = self.datas[0].datetime.date(0)
        log_msg = (
            f"Date: {current_date}, Close: {self.dataclose[0]:.2f}, "
            f"RSI: {self.rsi[0]:.2f}, MACD: {self.macd.lines.macd[0]:.2f}, "
            f"Signal: {self.macd.lines.signal[0]:.2f}, "
            f"Crossover: {self.macd_crossover[0]:.2f}, "
            f"Oversold Signal Active: {self.oversold_signal_active}"
        )
        print(log_msg)

        if self.order:
            return

        # --- 入场逻辑 ---
        if not self.position:
            # 步骤 1: 检查RSI是否进入超卖区，以激活“等待确认”状态
            if self.rsi[0] < self.p.rsi_oversold:
                self.oversold_signal_active = True
                print(f"Date: {current_date} - Oversold condition met (RSI < {self.p.rsi_oversold}). Waiting for MACD confirmation.")

            # 步骤 2: 如果“等待确认”状态已激活，则寻找MACD金叉作为买入信号
            # self.macd_crossover[0] > 0 表示当前K线发生金叉
            buy_condition = self.oversold_signal_active and self.macd_crossover[0] > 0

            # --- 核心准则：诊断日志 (入场条件判断) ---
            print(f"Date: {current_date} - Evaluating Buy. Buy Condition Met: {buy_condition}")

            if buy_condition:
                self.log(f'BUY CREATE, Close: {self.dataclose[0]:.2f}')
                # 计算下单手数
                cash = self.broker.get_cash()
                size = (cash * self.p.sizing_pct) / self.dataclose[0]
                size = math.floor(size) # 取整
                self.order = self.buy(size=size)
                # 下单后重置状态，避免连续下单
                self.oversold_signal_active = False
        
        # --- 出场逻辑 ---
        else:
            # 条件 1: RSI进入超买区
            exit_by_rsi = self.rsi[0] > self.p.rsi_overbought
            # 条件 2: 触发止损
            exit_by_sl = self.dataclose[0] < self.buyprice * (1.0 - self.p.stop_loss)
            # 条件 3: 触发止盈
            exit_by_tp = self.dataclose[0] > self.buyprice * (1.0 + self.p.take_profit)

            if exit_by_rsi:
                self.log(f'SELL CREATE (RSI Overbought), Close: {self.dataclose[0]:.2f}')
                self.order = self.close()
                # 交易结束后重置状态
                self.oversold_signal_active = False
            elif exit_by_sl:
                self.log(f'SELL CREATE (Stop Loss), Close: {self.dataclose[0]:.2f}')
                self.order = self.close()
                self.oversold_signal_active = False
            elif exit_by_tp:
                self.log(f'SELL CREATE (Take Profit), Close: {self.dataclose[0]:.2f}')
                self.order = self.close()
                self.oversold_signal_active = False

        # --- 核心指令：记录每日市值 ---
        self.daily_values.append(self.broker.getvalue())
